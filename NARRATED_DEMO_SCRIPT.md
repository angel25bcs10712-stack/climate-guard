# ClimateGuard Narrated Demo Script (2-3 Minutes)

## Recording Setup (Before You Start)
- Open `http://localhost:8000`
- Ensure `ollama serve` is running
- Keep audio clear and pace steady
- Use the built-in **Demo Walkthrough Mode**

## 0:00 - 0:20 | Opening
"This is ClimateGuard, an offline disaster preparedness assistant built for climate-vulnerable communities where internet and power may fail during emergencies. The app runs fully on-device using Gemma through Ollama, so it remains available when cloud tools are not."

## 0:20 - 0:40 | Problem and Why It Matters
"In many regions like rural Bangladesh, parts of Sub-Saharan Africa, and Pacific Island communities, disaster risk is high but connectivity is fragile. ClimateGuard focuses on that gap: practical, local, life-saving guidance with zero internet dependency."

## 0:40 - 0:55 | Product Overview
"Users can select a disaster type, enter location context, describe the situation, and optionally upload an image. The assistant always returns structured emergency guidance in four sections: Immediate Actions, Shelter, Supplies, and Contacts."

## 0:55 - 1:45 | Live Demo Walkthrough
Action:
- Click **Start 3-Scenario Demo**
- Let the scripted flow run

Voiceover:
"Now I am running the one-click demo walkthrough mode. It automatically executes three realistic scenarios: flood escalation, wildfire threat, and heatwave emergency."

"For each scenario, ClimateGuard generates structured response output. You can also see a severity estimate and response confidence indicator to make interpretation faster in high-stress situations."

## 1:45 - 2:10 | Incident Reporting
Action:
- Click **Export Incident Report (PDF)**
- Optionally click **Share Report**

Voiceover:
"After each response, ClimateGuard can generate an incident report that can be exported as PDF or shared directly. This helps responders, volunteers, and local coordinators pass actionable information quickly."

## 2:10 - 2:35 | Technical Credibility
"Technically, this project includes local multimodal support, function-calling integration, runtime guardrails for input and file validation, and operational endpoints for health and status checks. It is not a mockup - it is an end-to-end working system."

## 2:35 - 2:55 | Hackathon Fit
"ClimateGuard aligns with the Gemma 4 Good priorities: social impact, technical execution, clear use case, and accessibility in constrained environments. It also targets Global Resilience, plus Ollama and Unsloth special tracks."

## 2:55 - 3:00 | Close
"ClimateGuard brings practical AI safety guidance to places where connectivity cannot be assumed - turning local hardware into a resilient disaster support tool."
